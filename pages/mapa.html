AWS IAM

O que é IAM?

É o módulo da AWS para controlar usuários, permissões e acessos às instâncias, aplicações e serviços. Fundamental para segurança, pois evita usar a conta root (que tem acesso total, inclusive a billing/pagamentos).

Usuarios

São logins com usuário + senha e/ou chaves de acesso programáticas (API keys). Podem ser usados no console ou em aplicações. Problema: se credenciais ficarem no código (ex.: subir no GitHub), podem ser comprometidas.

Roles (funções)

Usuários especiais que não têm login direto. São usados para atribuir permissões a serviços AWS (ex.: EC2, Lambda, S3). Em vez de credenciais fixas, o serviço assume a role e herda suas permissões. Mais seguro do que espalhar chaves de acesso no código.

Grupos

Permitem organizar permissões para vários usuários ao mesmo tempo. Mudando a política do grupo, todos os usuários nele recebem a atualização.

Políticas (Policies)

Definem o que o usuário/role pode ou não fazer. Podem ser: Predefinidas pela AWS (ex.: AmazonS3FullAccess) ou Personalizadas (bem detalhadas, ex.: só leitura em um bucket específico).

DEVOPS E CLOUD

O que é DevOps?

DevOps é uma cultura e conjunto de práticas que busca integrar equipes de desenvolvimento e operação (e também teste, segurança, etc.).

Objetivo: entrega mais rápida, confiável e eficiente de software.

Vantagens de DevOps:
- Menor tempo de entrega (Time to Market)
- Reduz o tempo entre a ideia do produto e a entrega ao usuário.
- Mitigação de bugs
- Testes automatizados, monitoramento e feedback constante diminuem problemas.
- Maior confiabilidade e automação
- Automatiza processos repetitivos, liberando desenvolvedores para focar em valor.

Como DevOps ajuda na Cloud?

Infraestrutura como Código (IaC):
Ferramentas como Terraform, Crossplane, Pulumi, Ansible permitem criar e gerenciar infraestrutura programaticamente. Parametrização facilita replicar ambientes e controlar custos.

Contêineres (Docker, Kubernetes):
Isolam processos e simplificam ambientes de desenvolvimento e produção. Facilita gerenciamento de microsserviços e replicação de ambientes.

Pipelines e CI/CD:
Automatizam desde commits no Git até deploy em produção. Garantem testes, validação de segurança e análise de código sem intervenção manual.

Segurança e observabilidade:
Monitoramento e logs automatizados. Dashboards e alertas fornecem feedback contínuo para equipes, melhorando performance e experiência do usuário.

DevOps e Cloud

Apenas Cloud oferece resiliência e escalabilidade, mas ainda é manual e repetitivo. Apenas DevOps sem Cloud limita o alcance e a escalabilidade.

Juntos:
- Criação, gestão e deploy de infraestrutura se tornam ágeis, eficientes e escaláveis.
- Comunicação entre equipes melhora e feedback se torna contínuo.
- Profissional se torna mais diferenciado e preparado para qualquer projeto de TI.

Infrastructure as Code (IaC)

Modelos de Infraestrutura

Tradicional (On-Premises):
- Processos manuais (comprar hardware, instalar, cabeamento, configurar).
- Falta de documentação e histórico de mudanças.
- Alto custo (hardware, redundância, backup, refrigeração, licenças).
- Escalabilidade limitada, cara e demorada.
- Ambiente estático.

Cloud:
- Tudo virtualizado (hardware → software).
- Modelos de serviço: IaaS → AWS, Azure, GCP; PaaS → Kubernetes, Docker, GitHub, Salesforce; SaaS → Office 365, Google Docs, Dropbox, OneDrive.
- Automação com IaC.
- Recursos sob demanda → paga pelo uso.
- Escalabilidade e elasticidade.
- Baixo custo inicial, rápida implementação.
- Alta disponibilidade → multi-região, CDNs.

O que é IaC?

Definição: Infraestrutura criada e gerenciada como software.
Usa princípios do desenvolvimento de software + DevOps.
Baseada em CI/CD pipelines: integração contínua + entrega contínua.
Controle de versão (ex.: GitHub).
Ambientes replicáveis: Dev → QA → UAT → Produção.
Flexibilidade: criar/destruir ambientes em qualquer região.

Benefícios do IaC

- Automação
- Produtividade
- Validação e testes automatizados
- Colaboração
- Padronização
- Documentação embutida
- Auditoria fácil
- Redução de tarefas repetitivas

Cloud Security

O que é Cloud Security

Proteção da infraestrutura e aplicações na nuvem. Segurança deixa de ser física (data centers) → passa a ser configuração, acessos e permissões.

Responsabilidade Compartilhada

Segurança é dividida entre Cliente e Provedor de Nuvem. On-premise → responsabilidade total do cliente. IaaS → PaaS → SaaS → Functions → responsabilidades vão migrando para o provedor. Cliente foca em código, configurações e acessos.

Camadas de Segurança na Cloud

- IAM – identidades e acessos.
- Network – controle do tráfego.
- Endpoint – dispositivos e ativos que acessam.
- Aplicações – segurança em apps hospedados.
- Dados – proteção de bancos de dados e objetos.
- Logs & Monitoramento – visibilidade e auditoria.
- Organização – estrutura de contas/projetos/subscriptions.
